Aprende sobre GitHub

Si eres o formas parte de un equipo de desarrolladores, te gusta programacion, tienes amistades envueltas en este mundo, o simplemente estas interesado en aprender un nuevo hobby digital, posiblemente hayas escuchado sobre GitHub, y quieras conocer o reforzar tus conocimientos sobre esta plataforma. En esta recopilación de conceptos conocerás un poco más sobre la plataforma de desarrollo colaborativo más importante del mundo.

Nota importante (cuidado con olvidarlo): Git es la herramienta de control de versiones que usas en tu computadora (local), mientras que GitHub es el servicio en la nube donde alojas esos proyectos.

¿Qué es Git?

Es un sistema de control de versiones distribuido de código abierto. Se utiliza para rastrear y gestionar los cambios en un proyecto a lo largo del tiempo, permitiendo revertir a versiones anteriores y trabajar en diferentes ramas simultáneamente sin afectar el código principal (demasiado útil si eres de los que hace miss clic a cada rato).

¿Qué es GitHub?

Es una plataforma de alojamiento web para proyectos que utilizan el sistema Git. Además de guardar tu código, ofrece herramientas de red social y colaboración, como seguimiento de errores, gestión de tareas y revisiones de código. Un fun fact es que fue escrito originalmente en Ruby on Rails.

¿Cómo funciona GitHub?

GitHub funciona como un repositorio remoto central. El flujo típico consiste en trabajar en tu computadora (local), guardar tus avances y luego subirlos al servidor de GitHub (bien práctico, ¿no?). Esto permite que otros colaboradores descarguen tus cambios, los revisen y los fusionen con el proyecto global.

¿Qué es un repositorio?

También llamado repo, es el espacio digital donde se almacenan todos los archivos de un proyecto y el historial completo de sus cambios. Es la unidad básica de almacenamiento donde los desarrolladores gestionan el código fuente.

¿Qué es un commit?

Un commit es una "instantánea" o fotografía del estado de tu proyecto en un momento específico (piensa en una cámara). Se recomienda realizar confirmaciones con frecuencia, cada vez que completes una unidad lógica de trabajo o arregles un error.

¿Qué es una rama (Branch)?

Es un puntero móvil que apunta a una de las confirmaciones. Permite crear una línea de tiempo paralela para experimentar o añadir funciones sin alterar la línea principal. Aunque antes se usaba master, actualmente la rama por defecto en GitHub se llama main.

¿Qué es push?

Es el comando git push que envía tus commits locales desde tu computadora hacia el repositorio remoto en GitHub. Es la acción de publicar tus avances para que el resto del equipo pueda verlos (como subir una foto a tus estados de WhatsApp o historias de Instagram).

¿Qué es pull?

Es la combinación de dos acciones: git fetch (descargar datos) y git merge (integrarlos). Se usa para actualizar tu repositorio local con los últimos cambios que otros hayan subido al servidor remoto.

¿Qué es merge?

Es la operación de fusionar dos ramas diferentes. Permite tomar las líneas de desarrollo independientes y unificarlas en una sola rama (generalmente integrando una rama de función dentro de la rama main).
